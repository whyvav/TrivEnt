# TrivEnt ‚Äî Claude Code Project Guide

## Project Overview
Flutter ERP app for a brick manufacturing firm. App name: **TrivEnt**.
Covers inventory, sales, purchases, expenses, parties (customers/vendors), manufacturing (BOM + production), and reports.
Backend: **Firebase** (Firestore for data, Firebase Auth for login). Multi-platform Flutter (Web app currently with mostly mobile usage, Android apk later).

## Tech Stack
- Flutter / Dart
- Firebase Auth + Cloud Firestore
- `firebase_core`, `firebase_auth`, `cloud_firestore`, `firebase_storage` (likely)
- PDF generation via `lib/services/pdf_service.dart`

## Navigation Structure
Single `MainShell` widget in `lib/main.dart` with a side drawer (desktop/tablet) and bottom nav (mobile).
Screen index constants are defined at the top of `main.dart`:
- 0 Dashboard, 1 Inventory, 2 Sales, 3 Purchases, 4 Expenses, 5 Parties, 6 BOM, 7 Manufacture, 8 Reports
- 9 Payment In, 10 Sale Return, 11 Estimate, etc.

## Data Model Conventions
- **ItemModel** (`lib/models/item_model.dart`): unit field is `primaryUnit` (String), NOT `unit`. Also has optional `secondaryUnit` + `conversionFactor`. The `unit` key still exists in Firestore documents for legacy compat but the Dart field is always `primaryUnit`.
- `category` on ItemModel is either `'product'` or `'raw_material'`.
- `stockValue` getter uses `salePrice` for products, `purchasePrice` for raw materials.
- Payment invoices: Payment-In uses prefix `T` (e.g. T01), Payment-Out also uses `T` prefix.
- All Firestore timestamps are stored as ISO-8601 strings (via `toIso8601String()`), not Firestore Timestamps.

## Firestore Conventions
- **Composite indexes**: Any query combining multiple `.where()` clauses with `.orderBy()` requires a composite index in Firestore. When writing or modifying such queries, flag the need for a composite index ‚Äî missing indexes cause silent empty results, not errors.
- Collections (confirmed): `items`, `stock_transactions`, `sales`, `purchases`, `payments_in`, `payments_out`, `expenses`, `parties`, `bom`, `manufacture_records`.
- All docs use string IDs generated by Firestore (`.doc().id` pattern).

## UI Conventions
- **Chevrons in drawer/navigation**: Always place on the **trailing (right) edge** of the row. Use `Icons.chevron_right` rotating to `Icons.expand_more` when expanded.
- **FAB clearance**: Lists with a FAB must have bottom padding (typically `80px`) so the last item isn't hidden behind the FAB.
- Consistent card-based list items with `ListTile` or custom rows; match existing screen styles when adding new list views.
- Theme is defined in `lib/theme.dart` ‚Äî use `Theme.of(context)` colors, don't hardcode.
